syslog-ng

[syslog-ng]({{ project_url }}) allows you to flexibly collect, parse, classify, rewrite and correlate logs from across your infrastructure and store or route them to log analysis tools.

docker
docs
Setup

Edit /config/syslog-ng.conf to configure your logging sources and destinations. Note: As the application does not run as root you cannot listen on ports < 1024.

The application pid, control file, etc. are all kept in /config so when using tools such as syslog-ng-ctl you need to specify the path e.g. syslog-ng-ctl reload -c /config/syslog-ng.ctl

More info at syslog-ng.

# syslog-ng - Generated by SH-Apps-Meta

version: "3.8"
services:
  syslog-ng:
    image: linuxserver/syslog-ng
    container_name: syslog-ng
    restart: unless-stopped
    ports:
      #  Syslog UDP
      - 5514/udp:5514/udp
      #  Syslog TCP
      - 6601/tcp:6601/tcp
      #  Syslog TLS
      - 6514/tcp:6514/tcp
    volumes:
      #  Stores logs collected by the syslog-ng service
      - < /path/to/your/var/log>:/var/log
      #  Stores config and application files
      - < /path/to/your/config>:/config
    environment:
      #  User ID
      - PUID=100
      #  Group ID
      - PGID=100
      #  Specify a timezone to use EG Europe/London.
      - TZ=Europe/London
