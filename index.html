Self Hosted App Metadata

The purpose of this project is to maintain a consistent schema for many self hosted apps.

It can be used for creating "app stores", generating docker-compose configurations, building alternative configuration tools, etc.

FAQ

Why not just use docker-compose?
Docker-compose is perfect for running applications, but its schema is missing information necessary to fully run the application. This includes information required for reverse proxies, dashboards, database setup, etc. The good thing is our schema can be used to generate docker-compose files.
How to use it?
If you are a developer, get involved with the project on Github. If you are a user, you can browse the data to create a configuration for your own server.
